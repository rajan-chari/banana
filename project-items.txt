1) manual procedure to test tags?
   Start server: `source emcom/.venv/Scripts/activate && emcom-server`
   Then in another terminal:
   ```
   emcom -i /tmp/b.json inbox                          # see unread tag
   emcom -i /tmp/b.json tag <id> important working     # add tags
   emcom -i /tmp/b.json tagged important               # list by tag
   emcom -i /tmp/b.json untag <id> working             # remove tag
   emcom -i /tmp/b.json read <id>                      # removes unread
   emcom -i /tmp/b.json inbox --unread                 # should be empty
   ```
2) I want to start claudes in 2 subfolders .  alice and bob. I want them to use the tools we build to work on something together.  ideas?
   ans: Setup:
   - Create `alice/` and `bob/` subdirs. Each gets its own `CLAUDE.md` with persona + instructions.
   - Start emcom-server once (shared). Each Claude registers via `emcom -i identity.json register --name alice` (or bob).
   - Each Claude's CLAUDE.md on-load section: register with emcom, then `/loop 1m emcom inbox --unread` to poll for messages.
   - The emcom skill already handles send/reply/inbox — Claude can use it natively.

   Collaboration ideas:
   - **Code review**: Alice writes code, sends it to bob via emcom. Bob reviews, replies with feedback.
   - **Ping-pong programming**: Alice implements feature, sends to bob. Bob writes tests, sends results back.
   - **Research + synthesis**: Alice researches topic A, bob researches topic B. They email findings to each other, then synthesize.
   - **Debate/red-team**: Give them opposing positions on a design decision. They argue via email, converge on a recommendation.

   Key: each CLAUDE.md should say "check emcom inbox regularly" and "use emcom to communicate with the other agent". The `/loop` skill automates the polling.

3) If we had 'check emcom inbox' would it know to use the skill?
   ans: Yes. The skill description includes triggers "check email", "inbox", and "any email-related request." The word "emcom" plus "inbox" would match. Claude sees the skill list in every system reminder and matches user intent to skill descriptions. "check emcom inbox" would trigger `/emcom` which runs SKILL.md, which maps "inbox" → `emcom inbox`. For a `/loop` prompt like `/loop 1m check emcom inbox`, the loop fires the prompt and Claude would invoke the skill each time.
