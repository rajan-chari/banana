My current point of view

The refactor now feels ready to land as a coherent improvement. The structure is clearer, the earlier naming/type issues are fixed, and the riskiest moved behavior has direct tests.

I would not continue splitting the code just to make files smaller. The useful modularity is already in place: server composition is separated from route/runtime concerns, and session behavior has focused collaborators for hooks, checkpoints, emcom, heuristics, and injection verification.

The next review should focus on behavior and edge cases, not on further decomposition as a goal by itself.

Known follow-up I did not take here

I did not add admin/preferences route tests in this pass. Those endpoints touch user-home preferences state, so I would rather test them with an explicit mocked home/preferences setup than add a quick test that depends on the developer machine.

That is a reasonable follow-up, but I would not block this refactor on it.

Bottom line

The main open risk from the round-two review was missing focused tests. That has now been addressed with targeted controller coverage and a green build/test run. My recommendation is to treat this refactor as ready, with preferences-route tests as a later incremental hardening item.
